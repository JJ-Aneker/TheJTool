⚙ ¿Qué es el Category Builder?

El Therefore Category Builder es una herramienta web que permite diseñar visualmente las categorías de Therefore™ y exportarlas como un archivo XML listo para importar en Therefore Solution Designer.

Con él puedes:

  • Crear una o varias categorías en una sola sesión (multi-categoría)
  • Definir campos de datos, sus tipos y nombres de columna en base de datos
  • Organizar los campos en secciones con cabeceras de color
  • Activar pestañas con tabla de historial
  • Previsualizar el diálogo resultante antes de exportar
  • Importar estructuras desde un CSV o desde un XML de exportación real de Therefore
  • Guardar y reutilizar plantillas en la colección compartida
  • Aplicar paletas de colores corporativas en un clic
💡

El XML generado es compatible con el formato de importación nativo de Therefore Solution Designer v5.x / v6.x. No requiere instalación ni compilación.

🔄 Flujo de trabajo habitual

  1. Define el nombre visible y el TableName de la categoría en el panel Identidad.
  2. Elige la paleta de colores corporativa o personaliza los colores.
  3. (Opcional) Activa Pestañas si necesitas una tabla de historial en el diálogo.
  4. Añade secciones y dentro de cada una los campos que necesites.
  5. Haz clic en Generar XML → y descarga o copia el resultado.
  6. Abre Therefore Solution Designer, ve a Archivo → Importar configuración y selecciona el XML.
  7. Realiza los ajustes post-importación descritos al final de este manual.
ℹ️

También puedes empezar importando un CSV o un XML exportado desde otro servidor de Therefore — el builder extraerá automáticamente la estructura de campos.

🏷 Identidad de categoría

Cada categoría tiene tres atributos de identidad:

CampoDescripciónEjemplo
Nombre visible Nombre que verá el usuario en Therefore. Puede contener espacios y acentos. Facturas de Proveedor
TableName Nombre de la tabla SQL que creará Therefore. Solo letras, números y guion bajo. Si lo dejas vacío se genera uno automático. FACTURAS_PROVEEDOR
CtgryID Identificador único de la categoría en el XML, generado automáticamente a partir del nombre. Facturas_de_Proveedor
⚠

El TableName no puede coincidir con uno ya existente en Therefore. Si importas una categoría con el mismo TableName que otra existente, Therefore devolverá un error.

Multi-categoría

Puedes trabajar con varias categorías en paralelo usando las pestañas de categoría que aparecen encima del editor. El botón + Categoría añade una nueva. Al generar el XML, se incluyen todas las categorías en un único archivo de importación.

Los campos tipo Counter / AutoIncrement (TypeNo 9 en exports de Therefore) se convierten automáticamente a Texto (String) durante la importación XML.

📂 Secciones y campos

Los campos se agrupan en secciones, cada una con una etiqueta de cabecera de color. En el diálogo de Therefore se representan como barras de color que separan grupos lógicos de campos.

Crear y gestionar secciones

  • Usa el botón + Sección para añadir una nueva sección al final.
  • Edita el nombre directamente en el campo de texto (se convierte a mayúsculas automáticamente).
  • Elimina una sección con el botón ✕ Sección (solo aparece si hay más de una).

Crear y gestionar campos

  • Usa el botón + Campo dentro de cada sección para añadir un campo.
  • Nombre visible: texto que ve el usuario en el formulario.
  • ColName: nombre de la columna SQL (solo letras, números y guion bajo, máx. 30 caracteres). Se genera automáticamente a partir del nombre visible.
  • Tipo: tipo de dato (ver tabla anterior).
  • Longitud: solo visible para tipo Texto; indica el máximo de caracteres.

Layout visual

El builder coloca los campos de dos en dos por fila: el campo impar va a la izquierda y el par a la derecha. Usa el Preview para ver el resultado antes de exportar.

ℹ️

El campo ColName debe ser único dentro de la misma categoría. Therefore rechazará el XML si detecta dos campos con el mismo ColName.

📑 Pestañas e historial

Cuando activas la opción Pestañas, el diálogo de la categoría se divide en dos pestañas:

  • Pestaña 1 (configurable, por defecto "Datos"): contiene todos los campos y secciones que hayas definido.
  • Pestaña 2 (configurable, por defecto "Historial"): contiene una tabla vinculada a la tabla de historial TheIxTable_[Nombre]_Hist.
⚠

La tabla de historial TheIxTable_[Nombre]_Hist debe existir en Therefore para que la pestaña funcione. Si la categoría es nueva y no tiene historial previo, puedes activar la pestaña igualmente y configurarla después.

Columnas de la tabla (pestaña 2)

En el panel Columnas tabla defines las columnas que aparecerán en la tabla de historial. Cada columna tiene cabecera, ColName, tipo y ancho en píxeles.

📄 Importar campos desde CSV

Puedes importar la estructura de campos pegando un CSV directamente en el área de texto. Útil para preparar las categorías en Excel y pasarlas al builder.

Formato del CSV

Categoria;Nombre;Colname;Tipo;Long;Seccion
Facturas;Número factura;NUM_FACTURA;String;50;IDENTIFICACIÓN
Facturas;Fecha factura;;Date;;IDENTIFICACIÓN
Facturas;Importe total;;Money;;IMPORTES
Contratos;Número contrato;;String;50;IDENTIFICACIÓN
ColumnaObligatoriaDescripción
CategoriaNoSi se incluye, se crearán varias categorías separadas. Si se omite, todos los campos van a la categoría activa.
NombreSíNombre visible del campo.
ColnameNoColName SQL. Si se omite, se genera automáticamente desde el nombre.
TipoNoTipo del campo. Acepta alias: String, Texto, Date, Fecha, Integer, Money, Boolean, Timestamp, Lookup, Memo…
LongNoLongitud para campos Texto. Por defecto 100.
SeccionNoNombre de la sección a la que pertenece el campo. Por defecto "DATOS GENERALES".
💡

El separador puede ser ; o , — el builder lo detecta automáticamente. Los nombres de columna en la cabecera son insensibles a mayúsculas.

Modos de importación

  • Reemplazar: sustituye todo el contenido actual del builder por el CSV importado.
  • Añadir al final: agrega las secciones y campos del CSV a los existentes (las secciones con el mismo nombre se fusionan).

📥 Importar XML de Therefore

Si tienes un archivo XML exportado desde Therefore Solution Designer (exportación completa o parcial de configuración), puedes cargarlo directamente para extraer todas las categorías y sus campos.

Cómo importar

  1. Haz clic en Importar XML de Therefore en el panel izquierdo del editor.
  2. Pulsa Seleccionar archivo XML y elige el .xml exportado desde Therefore.
  3. El builder analiza el archivo y muestra un resumen: categorías encontradas, secciones y número de campos.
  4. Revisa las advertencias (si las hay) y elige Reemplazar todo o Añadir al final.

Qué se importa y qué se descarta

ElementoResultado
Campos de datos (ColName presente)Se importa
Secciones de color (TypeNo 4 con BClr)Se importa como sección
Etiquetas de diseño (TypeNo 4 sin color)Se descarta
Columnas de tabla relacionada (BelongsToTable)Se descarta
Campos vinculados (BelongsTo)Se descarta
Counter / AutoIncrement (TypeNo 9)Se importa como String
Tipos negativos (internos de Therefore)Se descarta
Workflows, Counters, Queries…Se descarta
ℹ️

El archivo XML puede ser una exportación completa del servidor (con workflows, formularios, etc.) o solo de categorías. El builder ignora todo lo que no sea categorías con campos de datos.

⚠

Las categorías que no tengan ningún campo de datos válido (solo etiquetas de diseño o campos de tabla) se omiten automáticamente y aparecen como advertencia.

☁ Colección de plantillas

Las plantillas permiten guardar y reutilizar configuraciones completas (una o varias categorías con sus campos) en la nube, accesibles desde cualquier sesión del portal.

Guardar una plantilla

  1. Haz clic en ⬆ Guardar como plantilla.
  2. Introduce un nombre (obligatorio) y descripción opcional.
  3. Pulsa Guardar. Se guardarán todas las categorías activas con sus campos.

Cargar una plantilla

  1. Haz clic en ▼ Colección de plantillas.
  2. Selecciona la plantilla deseada en el desplegable.
  3. Pulsa Cargar →. El contenido actual se reemplaza con la plantilla.
⚠

Cargar una plantilla reemplaza todas las categorías y campos actuales. Si tienes cambios sin guardar, guárdalos antes como otra plantilla.

Eliminar una plantilla

Selecciona la plantilla en el desplegable y pulsa 🗑 Eliminar. La eliminación es permanente.

📤 Generar y exportar el XML

Una vez configuradas las categorías, pulsa el botón Generar XML → en el panel derecho del editor.

El builder valida que:

  • Exista al menos una categoría con nombre y campos.
  • Si no se especificó TableName, se genera uno automático (TMP_XXXXX).

Opciones de exportación

  • 📋 Copiar XML: copia el XML al portapapeles.
  • ⬇ Descargar .xml: descarga el archivo con nombre [tablename]_therefore.xml.
  • 🔗 Nueva pestaña: abre el XML en una pestaña del navegador para inspeccionarlo.
💡

El contador de caracteres y categorías en el pie del área de texto te ayuda a verificar que el XML contiene lo esperado antes de importarlo.

🖥 Importar el XML en Therefore Solution Designer

  1. Abre Therefore Solution Designer con permisos de administrador.
  2. Conéctate al servidor de Therefore de destino.
  3. Ve al menú Archivo → Importar configuración…
  4. Selecciona el archivo .xml generado por el builder.
  5. En el asistente de importación, asegúrate de que las categorías aparezcan marcadas para importar.
  6. Pulsa Importar y espera a que finalice el proceso.
🚨

Importar en un servidor de producción sin previa validación en un entorno de pruebas puede causar interrupciones. Siempre prueba primero en un servidor de desarrollo o staging.

🔧 Pasos post-importación

Una vez importada la categoría en Therefore, hay algunos ajustes que debes realizar manualmente en Solution Designer:

1 · Mover la categoría a la carpeta correcta

Las categorías importadas aparecen en la raíz del árbol. Arrástralas a la carpeta del departamento o proyecto correspondiente dentro de la estructura de categorías.

2 · Añadir campo de imagen / archivo

El builder genera solo los campos de índice (metadatos). Necesitas añadir manualmente un campo ImageFld (campo de documento) a la categoría para que Therefore permita asociar documentos a los registros.

3 · Convertir contadores si es necesario

Si algún campo de tipo String debería ser un TextCounter (autonumérico), cámbialo en la pestaña del campo dentro de Solution Designer.

4 · Configurar Lookup (listas desplegables)

Los campos de tipo Lista desplegable (TypeNo 15) necesitan un diccionario de palabras clave asignado. Ve a la configuración del campo en Solution Designer y enlaza el diccionario correspondiente.

5 · Revisar permisos de categoría

Asigna los grupos de usuarios con acceso a la nueva categoría en la pestaña de permisos de Solution Designer.

6 · Configurar workflows (si aplica)

Si la categoría participa en procesos de aprobación o notificación, configura los workflows desde Solution Designer → Procesos de trabajo.

💡

Recuerda hacer un Deploy del servidor de Therefore después de guardar todos los cambios para que los clientes reciban la nueva configuración.

❓ Preguntas frecuentes

¿Puedo importar más de una categoría a la vez?

Sí. Usa las pestañas de categoría del builder para crear varias categorías y pulsa Generar XML →. El XML resultante contendrá todas las categorías y se importarán en bloque.

¿Qué pasa si dos categorías tienen el mismo TableName?

Therefore rechazará la importación de la segunda categoría con error de tabla duplicada. Asegúrate de que cada categoría tenga un TableName único en el servidor de destino.

¿El campo ColName puede tener acentos o espacios?

No. Los ColNames son nombres de columna SQL y solo admiten letras (A-Z), números y guion bajo. El builder limpia automáticamente caracteres no válidos.

¿Por qué el XML importado en Therefore no muestra las secciones de color?

Verifica que el TableName del XML no exista ya en el servidor — si Therefore detecta una categoría duplicada puede aplicar la importación parcialmente. También puede ser que la versión de Therefore Solution Designer no sea compatible (requiere v5.x o superior).

¿Puedo editar el XML manualmente antes de importarlo?

Sí, es XML estándar. Si necesitas ajustes avanzados (orden de campos específico, DisplayProp personalizado, etc.) puedes editar el archivo con cualquier editor de texto antes de importarlo.

¿La colección de plantillas es accesible para todos los usuarios del portal?

Sí, las plantillas son compartidas. Cualquier usuario con acceso al portal Aneker puede ver, cargar y eliminar plantillas. No hay aislamiento por usuario actualmente.
